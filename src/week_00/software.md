# Software

Reference links for the tools you'll be installing, configuring, and using throughout the summer.

- **[Slurm](https://slurm.schedmd.com)** — the workload manager/job scheduler that runs the cluster the workstation is integrated with. You won't need to administer it, but you'll want to know how it shapes what the workstation can and can't do.
- **[Wireshark](https://www.wireshark.org)** — the graphical packet analyzer you'll use to open and inspect capture files (see the [Week 2 hands-on lab](../week_02/code) for a first run-through).
- **[tcpdump](https://www.tcpdump.org)** — Wireshark's command-line cousin: capture and filter packets from the shell, useful when you're on a machine without a GUI or want something scriptable.
- **[Containerlab](https://containerlab.dev)** — emulates network devices and wires virtual links between them; this is what powers the device-emulation labs on the [lab server](./lab-servers).
- **[GNS3](https://www.gns3.com)** — a graphical network emulation platform for building and running virtual topologies, often alongside real vendor device images. Another option if you want a GUI-driven alternative to Containerlab for exploring topologies.
- **[Cisco Packet Tracer](https://www.netacad.com/courses/packet-tracer)** — Cisco's free, fully-userspace network simulator; a low-stakes way to get extra switching/routing practice outside the lab environment.
- **[Git](https://git-scm.com)** — the version control system the [project repo](./repo) lives in. If you're new to it, the official docs and book are the best starting point.
