# Week 2 — Resources

## Required Readings

- [Beej's Guide to Networking Concepts](https://beej.us/guide/bgnet0/html/) — intro chapters (encapsulation, OSI/TCP-IP, Ethernet, LANs). Python-based, ideal level for where you're starting.
- [OSI model — Wikipedia](https://en.wikipedia.org/wiki/OSI_model) — the direct reference for the 7-layer model itself: what each layer does, and how OSI's seven layers map onto the four-layer TCP/IP model you'll actually be working with all summer.
- [Computer Networks: A Systems Approach](https://book.systemsapproach.org/), Ch. 1 (Foundation) — the "why does a network need an architecture at all" framing that the rest of the course builds on.
- [Khan Academy: The Internet Protocols](https://www.khanacademy.org/computing/computers-and-internet/xcae6f4a7ff015e7d:the-internet/xcae6f4a7ff015e7d:the-internet-protocol-suite/a/the-internet-protocols) — a short, plain-language walkthrough of the layered protocol suite, written for readers with no networking background.

## Recommended Readings

- [GeeksforGeeks: OSI Model — 7 Layers Explained](https://www.geeksforgeeks.org/computer-networks/open-systems-interconnection-model-osi/) — walks a single email through all seven layers; a good companion to the Wikipedia reference if you want a worked example rather than a definition-by-definition read.
- [Baeldung: Circuit Switching vs. Packet Switching](https://www.baeldung.com/cs/circuit-switching-vs-packet-switching) — a clear side-by-side comparison that grounds *why* the Internet is packet-switched in the first place.
- [High Performance Browser Networking, Ch. 1 — Primer on Latency and Bandwidth](https://hpbn.co/primer-on-latency-and-bandwidth/) — free online; useful context for *why* the layered, packet-switched design has the performance characteristics it does (propagation, transmission, processing, and queuing delay).
- [Cornell CS 4410: Networking Overview & Link Layer (lecture notes)](https://www.cs.cornell.edu/courses/cs4410/2017su/lectures/lec19-linklayer.html) — a more rigorous, course-style treatment of the layered model and the link layer, if you want to go a level deeper than the intro readings.
